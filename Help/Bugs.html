Known Bugs/Issues

Template title disappearance

Intermittently, when viewing the Template File List, the template group title will be blank, but the path will still be filled in correctly. Sometimes the templates within that group work, sometimes they don't.

This is a bug, but it is rarely reproducible.

Resolution: Remove the template group in question and readd it.

Keyboard paste from clipboard doesn't work

This will remain documented here, although it has been determined this is not a Workspace Whiz problem. Instead, there seems to be a pattern of it only occuring when any macro is used.

Occasionally, there are intermittent problems pasting text into Developer Studio. This will only result when copying an entire line (not a portion of a line). The entire line can be copied to the clipboard and pasted into another app (like Notepad), but the paste function in Developer Studio will not work. Then, seemingly out of the blue, it works fine.

Microsoft's Knowledge Base has a variation of this problem. Article Q231652 describes the Paste command failing under NT4 when Service Pack 4 is applied.

Additional occurrences have been reported by Workspace Whiz users under Windows NT4 SP5 and SP6 and Windows 2000 Build 2195. All occurrences have happened with Visual C++ 6.0 SP3. No Visual C++ 5.0 reports have been made, although the Knowledge Base article confirms it happening for VC5.
