Template Files

Template Section: Dialog

The Dialog section is not required.

The Dialog section may contain one or more dialogs the end-user may read. The dialogs may also be used to solicit user input. The contents of the Dialog can be full HTML markup, including JavaScript or VBScript.

!!Dialog
<p>This is a paragraph test</p>
<p>This is a <b>bold, <i>italic</i></b> test.</p>
!!End

Dialog sections may contain multiple Next commands. A Next command exists on its own line and is preceded by two exclamation points.

!!Dialog
This is page 1.
!!Next
This is page 2.
!!Next
This is page 3.
!!End

The id parameter of an HTML tag determines the name of the template dictionary entry used to store the user input:

!!Dialog

[input type=checkbox id=dothirdparam name="checkbox"]Display third parameter?
!!End

In the above example, a dictionary entry called dothirdparam would be added to the dictionary.

The following HTML user input types are supported:

HTML input type Dictionary values
text A direct representation of the string typed.
file The full path name of the file selected.
checkbox "1" if the checkbox was checked, "0" if it wasn't.
radio "1" for the selected radio button, "0" for all others in the group.

Any other user input types are not translated into the dictionary.

Variable substitution may not be used in the Dialog section.
