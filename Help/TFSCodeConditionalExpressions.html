Template Section: Code: Conditional Expressions

The template system supports simple conditional expressions of the form if-elif-else-endif. In every respect, conditional expressions look just like code commands. They must exist on the first column of the line. They begin with double explanation points.

The simplest conditional expression is seen below:

!!If 0
Won't ever get here.
!!Endif

The above code won't ever evaluate, because the condition is always false.

The reverse case of this will always evaluate:

!!If 1
Gets here every time.
!!Endif

Using variables:

!!Set condition 1
!!If @@condition@@
Hello, world!
!!Endif

This tests condition against 0. If it isn't 0, then the conditional evaluates to true, and Hello, world! is printed.

It is important to use the variable form when specifying condition: @@condition@@. This substitutes the value of the variable, and that is what the condition expects.

Note that 0 is the only false value. While 1 is a true value so is 1000 or This is my string.

Alternatively, there is a similar test to ensure the conditional isn't true:

!!Set condition 1
!!If !@@condition@@
Hello, world!
!!Endif

Conditionals statements may take advantage of the else statement:

!!Set test 0
!!If @@test@@
Test is true.
!!Else
Test is false.
!!Endif

Since test isn't true (not equal to 0), Test is false. is displayed.

The elif statement makes conditional evaluation even more powerful.

!!Set test1 0
!!Set test2 1
!!If @@test1@@
Test1 is true.
!!Elif @@test2@@
Test2 is true.
!!Endif

Since test1 isn't true and test2 is, the text Test2 is true. is printed.

Full facilities for the == (is equal to), != (is not equal to), >= (greater than or equal to), <= (less than or equal to), > (greater than), and < (less than) conditional operators are available.

Format: !!if expr1 [cond] [expr2]

expr1 is always required in an !!If or !!Elseif statement. If cond exists (which is the ==, !=, >=, <=, >, or < symbols), then expr2 must exist also.

If expr1 and expr2 are both integers, they are evaluated as such. If expr1 or expr2 is a string, then both are evaluated as strings. For integers, this means that an operator such as >= will compare expr1 and expr2 as if they existed on the number line. For strings, the >= operator compares them alphabetically.

!!Set test1 cde
!!Set test2 abc
!!If @@test1@@ < @@test2@@
@@test2@@ is not less than @@test1@@.
!!Else
@@test2@@ is greater than @@test1@@.
!!Endif

Nested conditionals are also possible.

!!Set test1 1000
!!Set test2 500

!!Set test3 750
!!If @@test1@@ > @@test2@@
!!If @@test3@@ > @@test1@@
No way!
!!Endif
!!Endif
