Template Files

Commands

Template commands allow complex processes to be accomplished. There is, for instance, a command that creates a new file. There is another command that returns the name of current project.

Two types of commands exist. The first kind of command ignores return values (or has none). This type of command is specified by beginning a line with a double set of exclamation points. The command name and all related arguments follow.

!!// Specifies a comment, similar to the C++ // comment style.
!!FileNew MyNewFile.h dsp

The above example executes a command called FileNew that takes two arguments. The first argument is MyNewFile.h and the second argument is dsp. Arguments are separated by spaces, unless they are enclosed in quotation marks.

The second type of command accepts return values. This type of command uses an enhanced substitution expression syntax that begins with a @@( and ends with a )@@. Like normal substitution expressions, the result of the template command is substituted into the code being evaluated. Command substitution expressions may exist anywhere on a line.

The current project is @@(ProjectName)@@.

A command of the form @@(!!Command)@@ ignores the return value.

New commands may be added to Workspace Whiz's template processor. The API is described in the Workspace Whiz Source Code.
