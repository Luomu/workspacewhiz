Refreshing or Rebuilding Tags

In its default behavior, Workspace Whiz will automatically sense when a file has changed and update the tag list accordingly. Unfortunately, if the workspace contains hundreds of files, this detection process can be somewhat slow.

The Preferences Tags dialog offers a way to turn off the automatic refresh of tags. When a Workspace Whiz! tag command is issued, the tag command will potentially execute a lot faster.

The Find Tag dialog offers a Refresh Tags button. The Refresh Tags button does the same thing as the automatic refresh, but the operation is performed at user discretion. Tags generally don't need to be refreshed every time a file is saved, unless the changes made to the file alter attributes such as function signatures or global variables (or other items pertaining to the tag type in question). Pressing this button causes a Save All to be performed.

Although it should be rarely used, a Rebuild Tags button is also provided. Using it will completely rebuild the entire tags database.
