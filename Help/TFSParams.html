Template Section: Params

The Params section is not required.

The Params section contains any default variables used within the Code or Dialog sections.

!!Params
myVariable 1001
myVariable2 This is a test.
text1 @@(ProjectName)@@_TestClass
text2 @@(Env EAGLE)@@Hello
text3 a@@text1@@bc
!!End

This example shows several methods of specifying a variable.

The first line assigns the value 1001 to myVariable.

The second entry assigns the string This is a test. to myVariable2.

The third entry calls the template command ProjectName. Commands that return a value are specified in the @@(MyCommand)@@ substitution format. Just like variable substitution, variable expression commands can exist anywhere.

Since @@(ProjectName)@@ is a variable expression command, it returns a value. The contents of @@(ProjectName)@@ are replaced with the return value of ProjectName.

Any variables are evaluated immediately before the template is run. If there are user input dialogs, the variables are evaluated before the dialogs are processed.
