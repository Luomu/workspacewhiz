Template Variables and Substitution Expressions

The terms variable and dictionary entries are interchangeable. Both refer to the list of key-value pairs each template contains. Variables are used for counters, conditional statements, or as a substitution facility.

The substitution expression form is started with a @@ and ended with a @@. A simple substitution expression may appear as @@expression@@. The contained expression can be a variable or a template command call. The entire body, @@expression@@, is replaced with the result of expression's evaluation.

All variable expressions come in the substitution form:

@@MyVariable@@

When specified as above, the contents of MyVariable are inserted into the code in place of @@MyVariable@@.

Substitution expressions can be formatted in a variety of ways. To specify formatting, the expression looks like:

@@expression:[flags]@@

Flags are case sensitive and can be any of the following:

Flag Meaning
U Make the expression result uppercase.
L Make the expression result lowercase.
l

Left justify the expression result. A number, specifying the number of columns to justify the expression result within, must follow the flag.

@@expression:l##@@

If ## is less than the length of the result of expression, the result is truncated to ## characters.
r

Right justify the expression result. A number, specifying the number of columns to justify the expression result within, must follow the flag.

@@expression:r##@@

If ## is less than the length of the result of expression, the result is truncated to the right ## characters.
c

Align the expression result to the specified column.

@@expression:c##@@

For this flag, it isn't necessary to specify a variable or command in the expression:

@@:c70@@

It is entirely possible to combine two flags together:

@@expression:Ur60@@

This inserts the result of expression in uppercase, right justified within 60 spaces.

A specialized @@ command is the line continuation expression:

This is the first line -- @@\ this test is ignored.
this is continued on the first line.

The line continuation expression ignores the rest of the text on the current line, including the carriage return.

Predefined Variables

There are a couple predefined variables which may be used:

Variable Description
sel Set to the current text selection.
__Parent When the Call command is issued, __Parent of the callee is assigned to the caller.
__Return When the Call command is issued, using a Return command sets the callee variable __Return. The caller uses this variable to retrieve the return value.
