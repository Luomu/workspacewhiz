Extra Files

Overview

If there are external files (such as workspaces or projects) the application requires but whose files don't exist in the workspace, Workspace Whiz! allows those files to be transparently added via the Extra Files dialog. The Extra Files specified are used in the file finding and tag selection facilities.

Extra Files filenames can take advantage of environment variables. Environment variables specified as paths provide an excellent method to base project, workspace, or file paths from. The same environment variable on one developer's system can point to an entirely different location on another developer's system!

Filenames use the same syntax Visual Studio requires:

$(MYLIB)\Prj\MyLib.dsp

The Extra Files settings are stored in a file called ExtraFiles.WW. This file is stored in the same directory as the workspace.

Extra Files are also available when no workspace is open. Add them the same as when a workspace is open.
