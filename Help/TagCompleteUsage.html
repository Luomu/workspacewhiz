Tag Completion

Usage

When no completion has been performed yet, WWTagCompleteNext (and WWTagCompletePrev) finds the first tag which matches the partial tag. Thereafter, WWTagCompleteNext and WWTagCompletePrev move to the next and previous matching tag. When the end (or beginning) of the matched tag list is reached, the current tag wraps to the opposite end of the list.

The partial tag may include the / (forward slash) and * (asterisk) substring search operators.

If the partial tag has mixed case, then a case-sensitive search is performed. Otherwise, the search is not case-sensitive.

In the event too many tags are matched, WWTagCompleteRestore will restore the original partial tag. The partial tag may then be further completed by hand and the completion can be run again.

The WWTagCompleteDlg command shows a dialog of matching tags. The user may select a tag from the list to complete the partial tag.
