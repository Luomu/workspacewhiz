Matching Tags

As characters are typed in the Tag to match: field, tags are incrementally matched and displayed in the tag list. By default, the search is not case-sensitive and only alpha-numeric characters are counted. Non alpha-numeric characters can be typed, but they are ignored. (For example, if there is a tag called __ABORT, it is only necessary to type abort.)

Alternatively, a mixed case tag may be typed in. At this point, the search becomes case-sensitive and all characters are counted.

Use the slash key [/] or asterisk [*] at any point to perform a substring (wildcard) search. A substring search may be performed at any point. To match all tags that have the word window in them, type /window or *window. Similarly, to match all tags that start with a c and have the word window in them, type c/window.

Multiple substrings may be searched for by using more than one substring symbol. For example d/gra/man would match a symbol called DirectXGraphicsManager.

To begin matching a tag's parent tag, insert a semicolon [;] in the Tag to match: field. Parent searching comes in the form [tag][;parent]. That is, the tag is optional (as is the ;parent). Both may be searched for to better isolate a tag. Tag parents also support a substring search.

The use of the Parent: field has the same effect as typing ;parent in the Tag to match: field.

Additional search attributes may be used.
